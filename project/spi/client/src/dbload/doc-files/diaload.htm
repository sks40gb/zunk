DiaLoad Utility

Command Line Interface

Revised Version 1.01

 

Functional description:

 

The command line interface to the DiaLoad Utility allows non-graphical specification of a request to import image volumes into the DIA database.

 

Processing description:

 

DiaLoad imports an LFP cross reference file into the DIA Coding database and then imports the TIF image files into the DIA Image database.

 

It can be run in two steps, first importing the cross-reference file and then in a second execution importing the Image files.  Or, by selecting the command line option “—all” both cross-reference and then image data will be imported in single execution.

 

DiaLoad creates a DIA project and stores all the images on the image CD into the Image database.

 

If no Image Path is provided (--imagepath) DiaLoad requires an LFP importfile be provided on the command line.   If an Image Path is provided but, no importfile is provided, then DiaLoad will only attempt to load the images from the Image Path.

 

A project and a volume name are ALWAYS required, and an Image Import only step must be given valid, existing project and volume names. Presumably created in an earlier Cross-reference import step.

 

A log file is generated in the current directory if the “—log” and/or the “—debug” options are specified.  The file name is run.log.  

 

By default, DiaLoad assumes both the image database and the coding database are accessible via the “localhost” IP address on the machine where DiaLoad is executed.   However, the connection information:  IP address, port number, user id and password used to connect to these database servers can be changed via the command line to support loading to machines remote from the loading machine.  DiaLoad also supports the image database residing on a different server from the coding database.

 

 

Current Limitations:

 

  1. Supports only LFP format.

 

 

 

 

Command Line Syntax:

 

[] characters indicate optional syntax

{ | } charcters indicate either value accepted

“value” indicates a value must be input following the command indicator

 

 

java dbload.diaload [--log]

 

optional parameters:

... [--debug]

... [--replace ]

... [--append ]

... [--all]

... [ {-i | --imagepath  }  value ] 

... [{--codingport  } value]

... [{--codinguser  } value] 

... [{--codingpwd  } value]

... [{--imageport  } value] 

... [{--imageuser  } value]

... [{--imagepwd  } value]

... [{--codinghost  } value] 

... [{--imagehost  } value]

... [{--batchspan  } value]

 

Required parameters:

 

... {-p | --project   } value 

... {-v | --volume  } value

... {-t | --type  } value

... [{-x | --xref }importfile]*

 

* required if no imagepath or –all is not specified
